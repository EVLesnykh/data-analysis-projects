# Hotel Rating Prediction (EDA + ML)

## Описание проекта

Проект посвящён предсказанию рейтинга отелей на основе данных о бронированиях и отзывах пользователей.
Сценарий имитирует реальную задачу для компании Booking: выявление отелей с возможными накрутками рейтинга.

**Цель:** Построить модель, которая предсказывает оценку отеля (reviewer_score).
Если предсказания сильно отличаются от фактических значений, это может указывать на нечестное поведение отеля.

**Метрика:** MAPE (Mean Absolute Percentage Error).

## Данные
* hotels_train.csv — обучающая выборка
* hotels_test.csv — тестовая выборка
* submission.csv — пример файла для сабмишна

## Этапы работы
1. Загрузка и первичный анализ данных
* Изучены типы данных и количество пропусков.
Объединены обучающая и тестовая выборки для подготовки признаков.
2. Очистка данных
* Обработаны пропущенные значения (lat, lng).
* Удалены дубликаты.
* Преобразованы даты и числовые признаки.
* Текстовые отзывы приведены к удобной форме.
3. Исследовательский анализ данных (EDA) и генерация признаков
* Проанализировано распределение рейтингов и отзывов.
* Выделены новые признаки: наличие позитивных/негативных отзывов, месяц и квартал, популярные теги, кодирование городов и названий отелей.
* Категориальные признаки преобразованы в бинарные для обучения модели.
4. Отбор признаков
* Построена тепловая карта корреляций для выявления мультиколлинеарных признаков.
* Удалены избыточные признаки.
5. Обучение модели
* Данные разделены на тренировочную и тестовую выборку.
* Обучена модель RandomForestRegressor.
* Предсказан рейтинг для тестовой выборки.

## Результаты

В ходе работы над моделью для предсказания рейтинга отеля удалось получить MAPE = 15,96%, что означает, что в среднем предсказания отличаются от реальных значений примерно на 16%.

Чтобы повысить точность модели, можно:
1. Оптимизировать параметры RandomForestRegressor или попробовать градиентный бустинг (XGBoost, LightGBM);
2. Использовать NLP (извлечение ключевых слов из положительных и отрицательных отзывов, определение тональности (позитивная/негативная) каждого отзыва) для извлечения признаков из текстовых отзывов (positive_review, negative_review);
3. Провести кросс-валидацию для более стабильной оценки модели;
4. Создать новые признаки, например: средний рейтинг по городу или стране и т.д.;
5. Удалить или объединить сильно коррелирующие признаки, чтобы избежать избыточности.

Применение данных шагов поможет сделать модель более точной и надёжной, что важно для выявления нечестных отелей.
